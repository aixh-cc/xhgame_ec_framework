# 初衷

## 🎯 想解决的问题

### **1. 去除复杂的框架学习成本**  
❓ 框架太难学？  
💡 本 EC 框架只做两件事：`挂载` 与 `卸载`，容易上手。

### **2. DIY的您自己的框架**  
❓ 不想学习别人的框架了，想打磨自己的框架？  
💡 本框架所有的模块都可以替换成自己的,唯独架构动不了。

### **3. 一套代码同时开发 2D 和 3D 游戏**  
❓ 只做过 2D，不懂 3D？  
💡 构建单位工厂时选择2d或者3d工厂,不用改核心,轻松实现3d开发。

### **4. 终端开发 + 单元测试，ai时代必备技能**  
❓ 想用ai开发cocos？  
💡 支持终端使用bunjs运行开发，开发过程顺手完成单元测试。

### **5. 功能组件化，轻松在线移植**  
❓ 想一键使用别人的组件，不想移植过来还要修修改改适配自己的？  
💡 配合“xhgame在线游戏构建”,一键安装在线组件。

### **6. 数据驱动 + Cocos Component = 天然 ViewModel**  
❓ 在找好用的 MVVM 框架？  
💡 试试本框架的数据驱动开发，会少许多开发负担。

## 🎯 当前略有不足点

### 1、一套代码同时开发 2D 和 3D 游戏 
这部分代码还得等作者后续抽空做项目时补上，具体实现是在构建单位工厂时选择2d单位工厂或者是3d单位工厂

### 2、功能组件化，轻松在线移植 
框架使用人数少或者没有更多人参与组件制作时，这部分前期只能靠作者提供了。

### 3、ai驱动开发 
制作开发文档的目的是为了ai更好的阅读，以及如何引导ai开发还在摸索中
